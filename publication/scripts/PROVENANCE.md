below. The `.sh` scripts in this directory reproduce them; they invoke the pipelines by their
GitHub coordinates (never a local `main.nf`). The committed tables are the verbatim pipeline
outputs (raw, un-slimmed).

## Notes
- PanPep and the newer sequence model (TULIP-TCR) enter via `--custom_model_name panpep` /
  `--models ŌĆ”,tulip-tcr`. PanPep's `_panpep.tsv` outputs are supplied to the main run as the
  custom model (`--custom_model_training panpep_meta_dataset.csv`).
- `run_tcrpmhcbinding_tcren.sh` uses `samplesheet_no_FLCMKALLL.csv` (FLCMKALLL dropped from
  iggytop_unseen ŌĆö see the script header for why).
- The `conda activate t_hub` lines in the scripts are the machine's nextflow env; adjust for
  your host. Notebook execution uses the `consensus` env (see `../requirements.txt`).
- The mpredpan filenames are stored flat as `<ds>_mixtcrpred-pan_predicted.tsv` (the pipeline
  writes them under per-dataset subdirs; renamed on copy to keep `data/mixtcrpred_pan/` flat).
- `training_overview/` (Fig 2 dataset overview) was trimmed once from two large static source
  files that are **not** part of this repo (they live only in the original analysis dir,
  `rev1_p5_pae_vs_performance/`): `train_mhc1_complete_highfreq_peptides_flhcc.tsv` (the full
  tcrpHLA training set with AlphaFold PAE/pLDDT) and `tcrphla_db_curated_mhc1.tsv` (curated DB
  with epitope species). The trim kept, for `binder == 1` rows only:
  - `pos_training.tsv` = columns `mhc, peptide, binder, identifier, model_2_ptm_pae,
    model_2_ptm_plddt` (20,809 rows);
  - `species_lookup.tsv` = `identifier ŌåÆ "Epitope species"`, restricted to those identifiers
    (20,808 rows; the flhcc peptide `df356728-ŌĆ”` is absent from the DB and is labelled in the
    notebook). The one-shot trimmer referenced those absolute source paths, so it was run once
    and deleted ŌĆö the committed tables above are the sole reproducibility artifact.
